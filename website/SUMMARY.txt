PROJECT OVERVIEW
================================================================================

HandsSpeak is a comprehensive web-based platform designed to bridge the 
communication gap between Deaf and hearing communities through innovative 
technology. The platform combines real-time ASL gesture recognition, interactive 
learning modules, and text-to-speech capabilities to make American Sign Language 
accessible to everyone.

Mission Statement:
"Communicate Beyond Words" - Making ASL learning free, fast, and interactive 
for everyone while promoting accessibility and inclusion.

Core Philosophy:
- Communication is a fundamental human right
- Technology should enable, not complicate
- Learning should be engaging and stress-free
- Accessibility for all users regardless of ability

Target Audience:
- ASL learners (beginners to intermediate)
- Hearing individuals wanting to communicate with Deaf community
- Educators teaching inclusive communication
- Families with Deaf members
- Anyone interested in sign language

Key Statistics:
- 15,000+ Active Learners
- 250+ Gestures in Database
- 98% Learning Satisfaction Rate
- 6 Core Gestures with Real-time Recognition
- 26 Alphabet Signs with Video Tutorials


TECHNICAL ARCHITECTURE
================================================================================

Backend Technology Stack:
- Framework: Flask 2.3.3 (Python web framework)
- Computer Vision: OpenCV 4.8.1.78 (image processing)
- Hand Tracking: MediaPipe 0.10.14 (Google's ML solution)
- Machine Learning: TensorFlow 2.13.0 (gesture recognition model)
- Text-to-Speech: gTTS (Google Text-to-Speech)
- Audio Processing: Pygame (audio playback)
- Server: Gunicorn 21.2.0 (production WSGI server)
- Data Processing: NumPy 1.24.3

Frontend Technology Stack:
- HTML5 (semantic markup)
- CSS3 (glass morphism design system)
- JavaScript (ES6+, vanilla JS)
- Google Fonts: Poppins & Nunito
- Font Awesome 6.4.0 (icons)
- Canvas API (video processing & keypoint rendering)

Design System:
- Glass Morphism UI with backdrop blur effects
- Brown color palette (#4E342E, #C49A6C, #A1887F)
- Responsive design (7 breakpoints: 320px to 1920px+)
- Professional animations and transitions
- Accessibility-first approach (ARIA labels, screen reader support)

AI/ML Components:
- Custom-trained TensorFlow model for ASL gesture recognition
- MediaPipe Hands for real-time hand landmark detection
- Prediction smoothing algorithm (3-frame temporal smoothing)
- Confidence threshold system (70% default)
- 21-point hand landmark tracking (63 coordinates: x, y, z)

PAGE 2: TRANSLATE PAGE (Real-time ASL Translator)
================================================================================

Purpose:
The translate page is the core functionality of HandsSpeak, providing real-time
ASL gesture recognition and translation using webcam input. It leverages AI/ML
to detect hand signs and convert them to text with high accuracy.

Key Features:

1. Live Camera Feed System
   - Real-time webcam video streaming
   - Mirrored display for natural interaction (like a mirror)
   - Three-layer canvas system:
     * Base video layer (webcam feed)
     * Output canvas (processed video)
     * Keypoint canvas (hand landmark visualization)
   - FPS counter and processing time display
   - Status indicator (Ready/Active/Error states)

2. Hand Landmark Detection
   - MediaPipe Hands integration
   - 21-point hand landmark tracking
   - Real-time keypoint visualization on canvas
   - Mirrored coordinates to match video display
   - Single hand detection (max_num_hands=1)
   - Optimized settings:
     * Model complexity: 0 (fast processing)
     * Detection confidence: 60%
     * Tracking confidence: 50%

3. Gesture Recognition Engine
   - TensorFlow model with 6 supported gestures:
     * Hello (wave gesture)
     * Bye (wave gesture)
     * Yes (thumbs up)
     * No (thumbs down)
     * Thank You (prayer hands)
     * Perfect (OK sign)
   - Confidence threshold: 70% (adjustable)
   - Prediction smoothing: 3-frame temporal averaging
   - Real-time confidence display with color-coded bar
   - Low confidence filtering to reduce false positives

4. Translation Display
   - Current detected text in large, readable format
   - Gesture name with confidence percentage
   - Hands detected counter
   - Visual confidence badge with color coding:
     * Green: High confidence (>80%)
     * Yellow: Medium confidence (60-80%)

5. Text-to-Speech System

6. Translation History

7. Camera Controls

   - GET /contact : Contact page
   - GET /assets/<filename> : Static asset serving
   - POST /api/process_frame : Frame processing endpoint
   - POST /api/text_to_speech : TTS endpoint

2. MediaPipe Initialization
   - Hands detection with optimized settings
   - Retry mechanism (3 attempts)
   - Fallback handling if initialization fails
   - Test frame validation
   - Drawing utilities setup

3. Model Loading System
   - Multiple fallback paths for model files
   - Automatic model discovery
   - Class mapping JSON loading
   - Model validation and testing
   - Exponential backoff retry logic
   - Maximum 5 load attempts
   - Error logging and reporting

4. Frame Processing Pipeline
   - Base64 image decoding
   - Frame flipping (mirror effect)
   - BGR to RGB conversion
   - MediaPipe hand detection
   - Landmark extraction (21 points × 3 coordinates)
   - Landmark validation and normalization
   - Model prediction
   - Confidence thresholding
   - Prediction smoothing
   - Response JSON generation

5. Prediction Smoothing Algorithm
   - Temporal smoothing over 3 frames
   - Weighted confidence calculation
   - Majority voting for stability
   - 2-second history window
   - Fallback to current prediction if no consensus

6. Text-to-Speech System
   - gTTS integration for speech synthesis
   - Pygame mixer for audio playback
   - Request debouncing (2-second cooldown)
   - Duplicate request prevention
   - Background audio cleanup
   - Thread-safe TTS request tracking
   - Automatic file cleanup after playback
   - Error handling with retries

7. Performance Monitoring
   - Frame processing counter
   - Average processing time calculation
   - FPS estimation
   - Consecutive error tracking
   - Last successful frame timestamp
   - Performance statistics API

8. Thread Safety
   - Threading locks for shared resources:
     * history_lock (translation history)
     * tts_lock (TTS requests)
     * model_lock (model access)
   - Thread-safe data structures
   - Atomic operations for counters

9. Error Handling
   - Comprehensive try-catch blocks
   - Detailed error logging
   - User-friendly error messages
   - Graceful degradation
   - Automatic retry mechanisms
   - Fallback strategies

10. Logging System
    - File logging (asl_translator.log)
    - Console logging
    - UTF-8 encoding for Windows compatibility
    - Timestamp and log level tracking
    - Detailed debug information
